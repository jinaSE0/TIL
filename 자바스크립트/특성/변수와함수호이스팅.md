변수 호이스팅
```
console.log(vari) // undefined <-> 만약에 let으로 선언했으면 참조에러 였을것
var vari = "vari"
console.log(vari)
```
```
자바스크립트 엔진은 소스코드를 평가하는 과정과 실행하는 두 과정으로 나눠서 처리한다.
먼저 소스코드를 평가하는 십점에 선언된 변수를 모두 실행컨텍스트에 소스코드를 평가하는 시점에 모든 변수는 undefined가 할당된다 (초기화)
그리고 나서 소스코드를 실행하는데 이때 한줄씩 차례대로 코드가 실행된다.
할당이 되기전에 콘솔로그등의 코드를 만나면 미리 초기화 되어잇는 언디파인드가 출력된다
렛과 콘스트도 호이스팅이 일어나지만 언디파인드로 초기화되기 직전 티디지라는 일시적사각지대에 머물게 된다
따라서 참조에러가 발생한다 
결론 모든 변수 선언은 호이스팅이 발생한다
그러나 바키워드로 선언한 변수만 호이스팅이 발생하는것처럼 보인다
이때 변수를 참조하면 참조에러가 발생하기 때문이다
```
