하나의 프로그램은 다양한 모듈이 합쳐져서 만들어지는데 
자바스크립트의 런타임인 node.js에서는 남이 만들어놓은 모듈을 npm 모듈이라함

```
node.js 환경에서 외부 라이브러리를 다운로드하기 위해 다양한 방법이 존재하지만, 
그중 대표적인 것은 바로 npm입니다. npm은 Node Package Manager로 일종의 앱스토어입니다. 
필요한 모듈을 다운로드할 수 있는, 모듈들이 모여있는 모듈 스토어입니다.
cf) 리눅스의 패키지 매니저가 apt, macOS의 패키지 매니저가 brew

```

```
package.json에는 이 프로그램을 실행시키기 위해 필요한 모듈들이 무엇인지, 프로그램을 실행시키는 방법, 프로그램을 테스트하는 방법 등이 명시되어 있습니다.

헷갈리면 안 되는 점은 이 프로그램을 실행시키기 위해 필요한 실제 모듈은 따로 node_modules이라는 폴더에 저장된다는 것입니다. package.json에는 어떤 모듈인지만 적혀 있습니다. 

- devDependencies : 프로그램 실행과 관계없는 오로지 개발을 위해 필요한 디펜던시,
  예를 들면 코드 모양을 잡아주는lint나 테스팅 모듈처럼 실제 프로젝트 동작에 직접적으로 영향을 주지않는 모듈들을 명시함
- dependencies는 devDependencies와는 다르게, 이 프로젝트가 돌아가기 위해 반드시 필요한 모듈들이 무엇인지가 적혀 있습니다. 
- scripts :  CLI에서 사용 가능한 명령/ 실행은 npm run 스크립트이름
```
